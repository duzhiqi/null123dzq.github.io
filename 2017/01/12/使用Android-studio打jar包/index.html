在右侧边栏的Gradle点开，找到对应Moudle的gradle选项，然后Task->other->makeJar,然后双击makeJar运行。

在gradle完成运行后，Mould工程下面的build->libs下就找到所需要jar包。

异常

如果代码有不能编译通过的错误，或者Android SDK配置support包某些类不能兼容都会导致不能正确打出jar。

  • :app:lint的错误可能会发生，在stackoverflow找到的解释是说代码不对，但是不排除其他情况，可以在moudle下面的build.gradle的android节点下面加入如下,来忽略lint的错误。

  • 比如我自己犯的错误就是，jar里面再使用switch的使用case的key使用了R文件里面的id，造成无法正确打出jar包，解决办法是用 if else 替换 switch的用法。

  • :appComplieDebugJavaWithJavac 的错误。这个错误可能是由于使用了support-annotation而v4不对的，需要将compileSdkVersion 和buildToolsVersion改成23就可以了。
  • 如果还有其他错误，可以make project一遍也许就能发现错误。

关于android studio打jar包

实际上在make project完成的时候，class文件都已经编译完成并生成名为classes.jar的jar包，上述脚本只是将其导出了而已。直接到moudle下的build->intermediates->bundles->debug目录下就能直接找到，并不一定是用脚本可以直接取出。
